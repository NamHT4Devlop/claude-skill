🏢 Install “Spec Kit” (the /namht-* commands) on a company machine

A step-by-step, beginner-friendly guide for people new to Claude Code: copy the nam-claude-skill toolkit onto your company machine, install it safely (no code leaks, no impact on team repos), and use it right away: understand the codebase, build features, fix bugs, review, write docs.

⏱ ~15 minutes🔒 Enterprise-safe 🙈 Zero footprint on team repos🔑 No API key needed

0. Prerequisites

  • Claude Code installed & working: open a terminal and run claude --version to see a version number. (If /plugin is unknown → update Claude Code.)
  • git: git --version. Node.js (for map & HTML): node --version (ideally ≥ 18).
  • Your company's Claude plan is Team/Enterprise (a commercial plan — Anthropic does not train on your code). Ask your admin if unsure.
  • Access to the toolkit repo NamHT4Devlop/nam-claude-skill (public) — or copy the folder over yourself.
⚠️ Note on the Claude plan
Use this only on an organization plan (Team/Enterprise). Don't use a personal plan (Pro/Max) for company code — the data policy is different.

1. Get the toolkit onto your machine

Put it in a fixed location so you can later update with git pull. For example ~/nam-claude-skill:

# option 1: clone from GitHub (needs repo access)
git clone https://github.com/NamHT4Devlop/nam-claude-skill.git ~/nam-claude-skill

# option 2: if given a zip/folder → extract it into ~/nam-claude-skill

Below this folder is referred to as <TOOL>. Windows: replace ~ with %USERPROFILE%.

2. Install for yourself (one command)

This command creates symlinks for the skills/commands/agents into your personal ~/.claude folder → usable in every repo, and stored in no repo (the team sees nothing).

~/nam-claude-skill/scripts/personal-install.sh

It prints the number of skills/commands linked + a reminder of the git-guard config snippet (done in step 3). Because it's symlinked, later a cd ~/nam-claude-skill && git pull updates everything instantly.

💡
A skill auto-activates when you speak naturally (e.g. “review this file”), or type /namht-... directly. No API key needed — it runs on your own Claude Code.

3. Enable the safety guards — IMPORTANT on a company machine

Two things ensure “whatever you run can't harm team repos”:

A

Global gitignore — keep artifacts out of commits

Every knowledge-base/, spec-kit-sessions/, .codegraph/ is ignored by git machine-wide (no repo is modified):

touch ~/.gitignore_global
printf '%s\n' 'spec-kit-sessions/' 'knowledge-base/' '.codegraph/' 'CLAUDE.local.md' '.spec-kit/' >> ~/.gitignore_global
git config --global core.excludesfile ~/.gitignore_global
B

git guard — read/sync only, no pushing to team repos

personal-install.sh has already placed the guard script at ~/.claude/hooks/namht-git-guard.sh. Now “wire” it into Claude Code: open ~/.claude/settings.json and add the block below (merge it in, don't delete what's there):

{
  "permissions": {
    "deny": [
      "Bash(git remote add:*)", "Bash(git remote set-url:*)", "Bash(git remote remove:*)",
      "Bash(git reset --hard:*)", "Bash(git clean -f:*)", "Bash(git rebase:*)",
      "Bash(git commit --amend:*)", "Bash(git restore:*)", "Bash(git branch -D:*)",
      "Bash(git send-email:*)"
    ]
  },
  "hooks": {
    "PreToolUse": [
      { "matcher": "Bash", "hooks": [
        { "type": "command", "command": "~/.claude/hooks/namht-git-guard.sh", "timeout": 10 } ] }
    ]
  }
}

Result: allows fetch · pull · status · log · diff · add · commit; hard-blocks pushing to team repos + destructive commands (reset --hard, rebase…).

🎯 Tune the push whitelist for a company machine
Open ~/nam-claude-skill/hooks/git-guard.sh, the ALLOW_OWNER_RE line:
  • To let the AI push to your personal repos: change NamHT4Devlop to your personal GitHub username.
  • Safest (recommended at a company): block the AI from pushing anywhere — set ALLOW_OWNER_RE='__NONE__'. When you need to push, run it yourself in a terminal.
🔁 Reload
After editing settings.json, open /hooks once or restart Claude Code so the guard takes effect.

4. CodeGraph (recommended — helps the AI understand code fast, fewer tokens)

CodeGraph builds a local “code map” the AI queries over MCP instead of reading blindly. 100% local.

# install the CLI via npm
npm i -g @colbymchenry/codegraph
# wire it into Claude Code (global level) + turn off telemetry
codegraph install -t claude -l global -y
codegraph telemetry off
🏢 More enterprise-friendly
If your company tightens supply-chain controls: install via npm with a pinned version (npm i -g @colbymchenry/codegraph@<version>) or through an internal mirror.

5. First use on a repo

# 1) cd into THE project you want to work on
cd /path/to/your-project
# 2) (if using CodeGraph) build the index once — it auto-updates afterwards
codegraph init
# 3) open Claude Code
claude

Inside Claude Code, the first time on a repo create the Knowledge Base, then use:

/namht-scan                 # scan the repo → generate knowledge-base/ (run once)
/namht-ask how does the login flow work?
/namht-build add a forgot-password feature via email OTP
/namht-fix-bug the /tasks API returns 500 on an invalid status

Already have a knowledge-base/ from another machine? Just copy it into the repo to reuse it — no need to scan.

7. Verify it works

  • In Claude Code, type / → you see the /namht-* commands.
  • Run /namht-help → it lists commands + reports whether the repo has a KB.
  • git guard: printf '{"tool_input":{"command":"git push https://github.com/some-org/x"}}' | ~/.claude/hooks/namht-git-guard.sh → outputs JSON deny.
  • CodeGraph: type / or ask a question about the code → Claude uses it automatically (after codegraph init + reload).

8. Update / Uninstall

# update the toolkit
cd ~/nam-claude-skill && git pull        # symlinked, so it applies immediately
# uninstall (only removes symlinks in ~/.claude, touches no repo)
~/nam-claude-skill/scripts/personal-install.sh uninstall
# remove CodeGraph from Claude Code
codegraph uninstall

10. FAQ & enterprise safety

Is it safe to copy onto a company machine?

The toolkit code is purely local (reads/writes files), makes no network/exec/eval calls, and contains no secrets. Audit details + how to verify: see <TOOL>/SECURITY.md.

Is any code data sent anywhere?

The KB/CodeGraph stay local. When the AI reads code to answer, that snippet enters the LLM context — just like Claude reading a file normally, and already permitted by your organization (Team/Enterprise plans don't train on your data).

How do I make sure team repos aren't touched?

Global gitignore (step 3A) + the git guard (3B) + setting ALLOW_OWNER_RE='__NONE__' to forbid the AI from pushing anywhere. Even stronger: enable branch protection on the team GitHub repo.

Does it remember me between sessions?

Yes. Claude Code has auto-memory (per-project) + you can keep general preferences in ~/.claude/CLAUDE.md (loaded in every repo). Each repo's KB also grows over each build/rescan.
