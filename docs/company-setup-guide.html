0. Điều kiện cần

  • Claude Code đã cài & chạy được: mở terminal gõ claude --version ra số phiên bản. (Nếu lệnh /plugin báo lạ → cập nhật Claude Code.)
  • git: git --version. Node.js (cho map & HTML): node --version (nên ≥ 18).
  • Gói Claude của công ty là Team/Enterprise (gói thương mại — Anthropic không dùng code của bạn để train). Hỏi admin nếu chưa rõ.
  • Quyền truy cập repo công cụ NamHT4Devlop/claude-skill (private) — hoặc bạn tự copy thư mục sang.
⚠️ Lưu ý gói Claude
Chỉ dùng trên gói tổ chức (Team/Enterprise). Đừng dùng gói cá nhân (Pro/Max) cho code công ty — chính sách dữ liệu khác.

1. Lấy bộ công cụ về máy

Đặt ở một chỗ cố định để sau này cập nhật bằng git pull. Ví dụ ~/claude-skill:

# cách 1: clone từ GitHub (cần quyền truy cập repo)
git clone https://github.com/NamHT4Devlop/claude-skill.git ~/claude-skill

# cách 2: nếu được đưa file zip/thư mục → giải nén vào ~/claude-skill

Bên dưới gọi thư mục này là <TOOL>. Windows: thay ~ bằng %USERPROFILE%.

2. Cài cho riêng bạn (1 lệnh)

Lệnh này tạo symlink các skill/command/agent vào thư mục cá nhân ~/.claude → dùng được ở mọi repo, và không nằm trong repo nào (team không thấy gì).

~/claude-skill/scripts/personal-install.sh

Nó in ra số skill/command đã link + nhắc đoạn cấu hình rào git (làm ở bước 3). Vì là symlink nên sau này cd ~/claude-skill && git pull là cập nhật ngay.

💡
Skill tự kích hoạt khi bạn nói tự nhiên (vd “review file này”), hoặc gõ thẳng /namht-.... Không cần API key — chạy bằng chính Claude Code của bạn.

3. Bật rào an toàn — QUAN TRỌNG cho máy công ty

Hai việc giúp “chạy gì cũng không hại repo team”:

A

Global gitignore — không để dấu vết lọt vào commit

Mọi knowledge-base/, spec-kit-sessions/, .codegraph/ sẽ bị git bỏ qua trên toàn máy (không sửa repo nào):

touch ~/.gitignore_global
printf '%s\n' 'spec-kit-sessions/' 'knowledge-base/' '.codegraph/' 'CLAUDE.local.md' '.spec-kit/' >> ~/.gitignore_global
git config --global core.excludesfile ~/.gitignore_global
B

Rào git — chỉ đọc/đồng bộ, cấm push repo team

personal-install.sh đã đặt sẵn script chặn tại ~/.claude/hooks/namht-git-guard.sh. Giờ “lắp” nó vào Claude Code: mở ~/.claude/settings.json và thêm khối sau (gộp vào, đừng xoá phần đang có):

{
  "permissions": {
    "deny": [
      "Bash(git remote add:*)", "Bash(git remote set-url:*)", "Bash(git remote remove:*)",
      "Bash(git reset --hard:*)", "Bash(git clean -f:*)", "Bash(git rebase:*)",
      "Bash(git commit --amend:*)", "Bash(git restore:*)", "Bash(git branch -D:*)",
      "Bash(git send-email:*)"
    ]
  },
  "hooks": {
    "PreToolUse": [
      { "matcher": "Bash", "hooks": [
        { "type": "command", "command": "~/.claude/hooks/namht-git-guard.sh", "timeout": 10 } ] }
    ]
  }
}

Kết quả: cho phép fetch · pull · status · log · diff · add · commit; chặn cứng push repo team + các lệnh phá huỷ (reset --hard, rebase…).

🎯 Chỉnh whitelist push cho máy công ty
Mở ~/claude-skill/hooks/git-guard.sh, dòng ALLOW_OWNER_RE:
  • Cho AI push repo cá nhân của bạn: đổi NamHT4Devlop → username GitHub cá nhân của bạn.
  • An toàn nhất (khuyên cho công ty): chặn AI push mọi nơi — đặt ALLOW_OWNER_RE='__NONE__'. Khi cần push, bạn tự chạy trong terminal.
🔁 Nạp lại
Sau khi sửa settings.json, mở /hooks một lần hoặc khởi động lại Claude Code để rào có hiệu lực.

4. CodeGraph (khuyến nghị — giúp AI hiểu code nhanh, ít token)

CodeGraph dựng “bản đồ code” cục bộ để AI truy vấn qua MCP thay vì đọc mò. 100% local.

# macOS/Linux: cài CLI (tự bundle runtime)
curl -fsSL https://raw.githubusercontent.com/colbymchenry/codegraph/main/install.sh | sh
# nối vào Claude Code (mức global) + tắt telemetry
codegraph install -t claude -l global -y
codegraph telemetry off
🏢 Chuẩn enterprise hơn
Thay vì curl|sh, nếu công ty siết supply-chain: cài qua npm pin phiên bản (npm i -g @colbymchenry/codegraph@<version>) hoặc mirror nội bộ. Windows: dùng install.ps1.

5. Dùng lần đầu trên 1 repo

# 1) vào ĐÚNG project bạn muốn làm
cd /đường/dẫn/tới/project-cua-ban
# 2) (nếu dùng CodeGraph) dựng index 1 lần — tự cập nhật về sau
codegraph init
# 3) mở Claude Code
claude

Trong Claude Code, lần đầu với 1 repo hãy tạo Knowledge Base, rồi dùng:

/namht-scan                 # quét repo → sinh knowledge-base/ (chạy 1 lần)
/namht-ask luồng đăng nhập hoạt động thế nào?
/namht-build thêm tính năng quên mật khẩu qua email OTP
/namht-fix-bug API /tasks trả 500 khi status sai

Đã có knowledge-base/ sẵn từ máy khác? Cứ copy vào repo là dùng lại được, khỏi scan.

7. Kiểm tra đã chạy đúng

  • Trong Claude Code gõ / → thấy các lệnh /namht-*.
  • Chạy /namht-help → in danh sách lệnh + báo repo có KB hay chưa.
  • Rào git: printf '{"tool_input":{"command":"git push https://github.com/some-org/x"}}' | ~/.claude/hooks/namht-git-guard.sh → ra JSON deny.
  • CodeGraph: gõ / hoặc hỏi một câu về code → Claude tự dùng được (sau khi codegraph init + reload).

10. FAQ & an toàn enterprise

Copy vào máy công ty có an toàn không?

Code công cụ là thuần local (đọc/ghi file), không gọi mạng/exec/eval, không chứa secret. Chi tiết audit + cách tự kiểm: xem <TOOL>/SECURITY.md.

Dữ liệu code có bị gửi đi đâu không?

KB/CodeGraph nằm local. Khi AI đọc code để trả lời, đoạn đó vào ngữ cảnh LLM — giống như Claude đọc file bình thường, và đã được tổ chức cho phép (gói Team/Enterprise không train trên data của bạn).

Làm sao chắc chắn không đụng repo team?

Global gitignore (bước 3A) + rào git (3B) + đặt ALLOW_OWNER_RE='__NONE__' để cấm AI push mọi nơi. Mạnh hơn nữa: bật branch protection trên GitHub repo team.

Nó có nhớ tôi giữa các phiên không?

Có. Claude Code có auto-memory (per-project) + bạn có thể để sở thích chung trong ~/.claude/CLAUDE.md (load mọi repo). KB của mỗi repo cũng lớn dần qua mỗi lần build/rescan.
