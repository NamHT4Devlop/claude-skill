📋 Cài thủ công — không clone, không tải

Dành cho máy không được git clone hay tải gì, và không chép file qua được (USB/ổ chia sẻ). Cách làm: dán 1 script để tạo khung file rỗng, rồi copy nội dung từng file từ GitHub (Raw) vào editor, cuối cùng chạy build/cài đặt.

🙅 Không clone🙅 Không tải ✍️ Dán nội dung tay📁 Đặt ở đâu cũng được 🔑 Không cần API key
📁 Đặt project ở ĐÂU CŨNG ĐƯỢC

Chỉ cần đổi biến ROOT thành đường dẫn bạn muốn — ví dụ ~/tools/nam-claude-skill, /opt/nam-claude-skill, hay một ổ khác. Toolkit chạy được từ mọi vị trí vì personal-install.sh tự xác định đường dẫn của chính nó và chỉ tạo symlink vào ~/.claude (nơi Claude Code đọc). Mọi lệnh bên dưới đều dùng "$ROOT" nên đổi 1 chỗ là xong.

💡 Mẹo giúp đỡ ~30 lần copy

Các file trong skills/*/references/ (html-builder.js, render-html.cjs, review-skills-universal.md, kb-steps.md) chỉ là bản sao sinh ra từ resources/ bằng scripts/sync-bundles.sh. Nên bạn chỉ copy 4 file gốc trong resources/ một lần, rồi chạy sync-bundles.sh để nó tự nhân ra mọi skill. Bạn chỉ phải dán tay ~59 file thay vì ~88.

Các bước
  1. 1. Tạo cây thư mục & file rỗng (dán 1 lần)
  2. 2. Copy nội dung từng file từ GitHub
  3. 3. Build / Cài đặt sau khi copy xong
  4. 4. Không dùng CodeGraph (vẫn chạy đủ)
  5. 5. Lưu ý quan trọng

🧱 1. Tạo cây thư mục & file rỗng

Dán đoạn này 1 lần vào terminal máy đích. Nó tạo đúng cây thư mục + file rỗng (bạn điền nội dung ở bước 2). Các file reference được cố tình bỏ qua — sync-bundles.sh sẽ sinh chúng ở bước 3.

ROOT=~/nam-claude-skill          # ⬅ đổi thành BẤT KỲ đường dẫn nào bạn muốn
mkdir -p "$ROOT"/{commands,agents,hooks,scripts,resources}

# 20 skill → mỗi skill 1 file SKILL.md
for s in namht-ask namht-build namht-design-review namht-discover \
         namht-document namht-fix-bug namht-map namht-pdf namht-plan namht-plan-review \
         namht-pr namht-qa namht-qa-integration namht-rescan namht-retro namht-review \
         namht-scan namht-security-audit namht-skillify namht-system-map; do
  mkdir -p "$ROOT/skills/$s"; touch "$ROOT/skills/$s/SKILL.md"
done

# reference RIÊNG (không tự sync) — chỉ map & pdf
mkdir -p "$ROOT/skills/namht-map/references" "$ROOT/skills/namht-pdf/references"
touch "$ROOT/skills/namht-map/references/"{build-map.cjs,graph-builder.js,viewer-template.html}
touch "$ROOT/skills/namht-pdf/references/html-to-pdf.sh"

# 4 file gốc (sync-bundles sẽ nhân ra các skill khác)
touch "$ROOT/resources/"{html-builder.js,render-html.cjs,review-skills-universal.md,kb-steps.md}

# 22 command
for c in ask build codegraph design-review discover document fix-bug help map pdf \
         plan-review plan pr qa-integration qa rescan retro review scan security-audit \
         skillify system-map; do touch "$ROOT/commands/$c.md"; done

# 7 agent
for a in namht-architecture-reviewer namht-business-consistency-reviewer \
         namht-business-flow-tracer namht-codebase-analyzer namht-impact-detector \
         namht-performance-reviewer namht-security-reviewer; do touch "$ROOT/agents/$a.md"; done

# hook + script
touch "$ROOT/hooks/git-guard.sh" "$ROOT/scripts/personal-install.sh" "$ROOT/scripts/sync-bundles.sh"

echo "✅ Tạo khung xong. Danh sách file cần dán nội dung:"; find "$ROOT" -type f | sort

📥 2. Copy nội dung từng file từ GitHub

Mở từng file trên GitHub, bấm nút Raw, chọn hết, copy, rồi dán vào đúng file rỗng tương ứng. Raw cho text thuần (không dính HTML). Mẫu URL:

https://raw.githubusercontent.com/NamHT4Devlop/nam-claude-skill/main/<đường-dẫn>

Ví dụ: .../main/skills/namht-build/SKILL.md, .../main/commands/build.md, .../main/hooks/git-guard.sh.

Phải dán (59 file): 20 × skills/*/SKILL.md · 21 × commands/*.md · 7 × agents/*.md · 4 × resources/* · hooks/git-guard.sh · scripts/personal-install.sh · scripts/sync-bundles.sh · 4 file riêng của map/pdf.

Tùy chọn / bỏ đượcLý do
skills/namht-map/references/graph-builder.jsFile lớn (compiled) — chỉ cho /namht-map. Không cần map thì bỏ luôn 3 file của map.
vendor/*.min.jsRender diagram offline. Bỏ được nếu máy có mạng (diagram tự dùng CDN). Air-gapped + cần diagram mới phải copy 2 file minified (rất to).
README / SECURITY / LICENSE / docs / tests / .github / .claude-plugin / hooks.jsonKhông cần để chạy bản cài cá nhân.

🔧 3. Build / Cài đặt (sau khi đã dán xong hết)

ROOT=~/nam-claude-skill          # ⬅ cùng đường dẫn đã chọn ở bước 1

# 1) cấp quyền chạy (file tạo bằng editor KHÔNG có execute bit — bắt buộc)
chmod +x "$ROOT"/scripts/*.sh "$ROOT"/hooks/git-guard.sh \
         "$ROOT"/skills/namht-pdf/references/html-to-pdf.sh 2>/dev/null

# 2) nhân các file reference dùng chung ra mọi skill (từ resources/)
cd "$ROOT" && bash scripts/sync-bundles.sh

# 3) cài cho riêng bạn (symlink vào ~/.claude; command tự thêm tiền tố namht-)
bash "$ROOT/scripts/personal-install.sh"

# 4) chặn rò rỉ: global gitignore
touch ~/.gitignore_global
printf '%s\n' 'spec-kit-sessions/' 'knowledge-base/' '.codegraph/' 'CLAUDE.local.md' '.spec-kit/' >> ~/.gitignore_global
git config --global core.excludesfile ~/.gitignore_global

5) Lắp rào git

Mở ~/.claude/settings.json và gộp khối sau vào (đừng xoá phần đang có):

{
  "permissions": { "deny": [
    "Bash(git remote add:*)", "Bash(git remote set-url:*)", "Bash(git remote remove:*)",
    "Bash(git reset --hard:*)", "Bash(git clean -f:*)", "Bash(git rebase:*)",
    "Bash(git commit --amend:*)", "Bash(git restore:*)", "Bash(git branch -D:*)", "Bash(git send-email:*)"
  ] },
  "hooks": { "PreToolUse": [
    { "matcher": "Bash", "hooks": [
      { "type": "command", "command": "~/.claude/hooks/namht-git-guard.sh", "timeout": 10 } ] }
  ] }
}
🎯 An toàn nhất trên máy công ty
Mở "$ROOT"/hooks/git-guard.sh và đặt ALLOW_OWNER_RE='__NONE__' để cấm AI push đi bất kỳ đâu. Sau đó restart Claude Code để rào có hiệu lực.

6) Kiểm tra

ls -la ~/.claude/skills | grep namht-     # thấy 20 skill
# rào git → phải in ra JSON "deny"
printf '{"tool_input":{"command":"git push https://github.com/some-org/x"}}' | ~/.claude/hooks/namht-git-guard.sh

Trong Claude Code gõ / → thấy /namht-*; chạy /namht-help.

🧭 4. Không dùng CodeGraph (vẫn chạy đủ)

CodeGraph là tùy chọn — guide này vốn đã không cài nó. Các skill tự phát hiện "không có .codegraph/" và chuyển sang dùng Read/Grep/Glob + Knowledge Base. Không cần cấu hình gì thêm; bỏ qua mọi bước liên quan codegraph là xong.

CodeGraph cung cấpThay bằng (đã có sẵn)
Tìm source + call-path nhanhRead/Grep/Glob built-in của Claude Code
Blast radius (ai gọi cái gì), consumer ẩnKnowledge Base (10-core-flows, 13-business-rules, 14-integrations, modules/) + grep tìm caller
Đồ thị code cho agent (MCP)/namht-map — đồ thị phụ thuộc HTML cho người xem (offline, KHÔNG cần CodeGraph; cần Node + 3 file của map)
✅ Nên làm khi thiếu CodeGraph
  • Đầu tư KB: chạy /namht-scan 1 lần/repo, /namht-rescan sau khi code đổi — KB là nền tảng chính thay cho CodeGraph.
  • Xem phụ thuộc: dùng /namht-map (nhớ copy graph-builder.js + build-map.cjs + viewer-template.html ở bước 2).
  • Khi sửa 1 hàm: chủ động grep tên hàm/endpoint để tìm nơi gọi.
⚠️ Điểm yếu cần biết
Phân tích blast-radius tự động yếu hơn (dễ sót consumer ẩn / dynamic dispatch) ở /namht-qa và /namht-build. Bù bằng KB đầy đủ + /namht-map + grep thủ công.

⚠️ 5. Lưu ý quan trọng

  • Dán nguyên văn — giữ nguyên dòng đầu (vd #!/usr/bin/env bash trong git-guard.sh, và các file .cjs/.js/.sh). Sai 1 ký tự đầu là script lỗi.
  • Execute bit không được giữ khi bạn tạo file bằng editor — bước chmod +x ở mục 3 là bắt buộc.
  • Không có git pull (bạn không clone). Muốn cập nhật: dán lại nội dung các file đã đổi, rồi chạy lại sync-bundles.sh + personal-install.sh.
  • Chính sách công ty: dựng lại tool ngoài trên máy công ty vẫn có thể thuộc diện quản lý của IT/Security — xác nhận trước khi dùng.
