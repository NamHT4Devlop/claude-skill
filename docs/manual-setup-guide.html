📋 Manual setup — no clone, no download

For a machine where you may not git clone or download anything, and may not transfer files (USB/share). The plan: paste one script to scaffold empty files, then copy each file's content from GitHub (Raw) into your editor, then run the build/install steps.

🙅 No clone🙅 No download ✍️ Paste content by hand🔑 No API key
💡 The shortcut that saves ~30 copies

Files under skills/*/references/ (html-builder.js, render-html.cjs, review-skills-universal.md, kb-steps.md) are just copies generated from resources/ by scripts/sync-bundles.sh. So you copy the 4 source files in resources/ once, then run sync-bundles.sh to fan them out to every skill. You only hand-paste ~61 files instead of ~90.

Steps
  1. 1. Scaffold the folders & empty files (one paste)
  2. 2. Copy each file's content from GitHub
  3. 3. Build / Install after copying
  4. 4. Important notes

🧱 1. Scaffold the folders & empty files

Paste this once into the target machine's terminal. It creates the exact tree and empty files (you fill them in step 2). The reference copies are intentionally left out — sync-bundles.sh generates them in step 3.

ROOT=~/nam-claude-skill
mkdir -p "$ROOT"/{commands,agents,hooks,scripts,resources}

# 21 skills → one SKILL.md each
for s in namht-ask namht-build namht-codegraph namht-design-review namht-discover \
         namht-document namht-fix-bug namht-map namht-pdf namht-plan namht-plan-review \
         namht-pr namht-qa namht-qa-integration namht-rescan namht-retro namht-review \
         namht-scan namht-security-audit namht-skillify namht-system-map; do
  mkdir -p "$ROOT/skills/$s"; touch "$ROOT/skills/$s/SKILL.md"
done

# unique references (NOT auto-synced) — map & pdf only
mkdir -p "$ROOT/skills/namht-map/references" "$ROOT/skills/namht-pdf/references"
touch "$ROOT/skills/namht-map/references/"{build-map.cjs,graph-builder.js,viewer-template.html}
touch "$ROOT/skills/namht-pdf/references/html-to-pdf.sh"

# 4 canonical files (sync-bundles fans these out to the other skills)
touch "$ROOT/resources/"{html-builder.js,render-html.cjs,review-skills-universal.md,kb-steps.md}

# 22 commands
for c in ask build codegraph design-review discover document fix-bug help map pdf \
         plan-review plan pr qa-integration qa rescan retro review scan security-audit \
         skillify system-map; do touch "$ROOT/commands/$c.md"; done

# 7 agents
for a in namht-architecture-reviewer namht-business-consistency-reviewer \
         namht-business-flow-tracer namht-codebase-analyzer namht-impact-detector \
         namht-performance-reviewer namht-security-reviewer; do touch "$ROOT/agents/$a.md"; done

# hooks + scripts
touch "$ROOT/hooks/git-guard.sh" "$ROOT/scripts/personal-install.sh" "$ROOT/scripts/sync-bundles.sh"

echo "✅ Scaffold done. Files to fill with content:"; find "$ROOT" -type f | sort

📥 2. Copy each file's content from GitHub

Open each file on GitHub, click the Raw button, select all, copy, and paste into the matching empty file. Raw gives plain text (no HTML wrapping). URL pattern:

https://raw.githubusercontent.com/NamHT4Devlop/nam-claude-skill/main/<path>

e.g. .../main/skills/namht-build/SKILL.md, .../main/commands/build.md, .../main/hooks/git-guard.sh.

🔧 3. Build / Install (after all files are filled)

ROOT=~/nam-claude-skill

# 1) make scripts executable (editor-created files have no execute bit — required)
chmod +x "$ROOT"/scripts/*.sh "$ROOT"/hooks/git-guard.sh \
         "$ROOT"/skills/namht-pdf/references/html-to-pdf.sh 2>/dev/null

# 2) fan out the shared reference files into every skill (from resources/)
cd "$ROOT" && bash scripts/sync-bundles.sh

# 3) install for you (symlinks into ~/.claude; commands get a namht- prefix)
bash "$ROOT/scripts/personal-install.sh"

# 4) prevent leaks: global gitignore
touch ~/.gitignore_global
printf '%s\n' 'spec-kit-sessions/' 'knowledge-base/' '.codegraph/' 'CLAUDE.local.md' '.spec-kit/' >> ~/.gitignore_global
git config --global core.excludesfile ~/.gitignore_global

5) Arm the git guard

Open ~/.claude/settings.json and merge this block in (don't delete what's there):

{
  "permissions": { "deny": [
    "Bash(git remote add:*)", "Bash(git remote set-url:*)", "Bash(git remote remove:*)",
    "Bash(git reset --hard:*)", "Bash(git clean -f:*)", "Bash(git rebase:*)",
    "Bash(git commit --amend:*)", "Bash(git restore:*)", "Bash(git branch -D:*)", "Bash(git send-email:*)"
  ] },
  "hooks": { "PreToolUse": [
    { "matcher": "Bash", "hooks": [
      { "type": "command", "command": "~/.claude/hooks/namht-git-guard.sh", "timeout": 10 } ] }
  ] }
}
🎯 Safest on a company machine
Open "$ROOT"/hooks/git-guard.sh and set ALLOW_OWNER_RE='__NONE__' to forbid the AI from pushing anywhere. Then restart Claude Code so the guard loads.

6) Verify

ls -la ~/.claude/skills | grep namht-     # 21 skills appear
# git guard → should print JSON "deny"
printf '{"tool_input":{"command":"git push https://github.com/some-org/x"}}' | ~/.claude/hooks/namht-git-guard.sh

In Claude Code, type / → you see /namht-*; run /namht-help.

⚠️ 4. Important notes

  • Paste content verbatim — keep the first line intact (e.g. #!/usr/bin/env bash in git-guard.sh, and the .cjs/.js/.sh files). A wrong first character breaks the script.
  • Execute bit is not preserved when you create files in an editor — the chmod +x in step 3 is required.
  • No git pull updates (you didn't clone). To update, re-paste the changed files' content, then re-run sync-bundles.sh + personal-install.sh.
  • Company policy: recreating an external tool on a work machine may still fall under IT/security rules — confirm before relying on it.
